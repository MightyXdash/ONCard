# ONCard v1.1.5

This is a focused stability and performance update.

## Fixed

- Hardened JSON to SQLite migration for upgraded users, including legacy subject files outside the current taxonomy.
- Fixed unsafe SQLite backup behavior under WAL mode by switching backups to SQLite's backup API.
- Fixed updater validation around malformed release metadata, helper launch failures, and invalid silent-patch installer paths.
- Fixed reinforcement and temporary embedding flows that could stall behind nested local event loops.

## Improved

- Reduced semantic-search overhead by replacing full embedding-cache deep copies with direct record lookups.
- Reduced startup warmup overhead by removing fake parallelism on a lock-serialized data path.
- Removed extra artificial delay from seed-card embedding/session-prep flow.
- Tightened release/update safety for patch delivery.

## Notes

- Existing user data is preserved.
- This release is intended to feel faster and more reliable on larger libraries.
